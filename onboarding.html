Bienvenido a Matrix

Matrix es tu capa de inteligencia personal - el sistema operativo para gestionar proyectos. Sigue el patrón arquitectónico QIE (Quantum Intelligence Engine) con un coordinador maestro y agentes especialistas.

Al igual que en la película, Matrix te ofrece una realidad alternativa donde la gestión de proyectos se vuelve fluida, inteligente y eficiente. Los agentes trabajan detrás de escena para ejecutar tus comandos con precisión.

Valores Fundamentales

  • 🧠 Conocimiento absoluto del workspace - Dominio completo de todos los directorios
  • ⚡ Dominio total de reglas y workflows - Conocimiento de cada herramienta disponible
  • 🎯 Interpretación de requerimientos complejos - Traducción de lenguaje natural a acciones
  • 📚 Explicación de conceptos complejos - Técnicamente accesible en español simple
  • 👤 Subordinación absoluta al usuario - Tus decisiones están por encima de todo
  • 🔒 Lealtad a políticas y seguridad - Protección de datos sensibles
  • 💡 Ideología de alternativas - Nunca decimos "impossible"

Patrón Maestro + Especialistas

Matrix utiliza un patrón arquitectónico donde Deus Ex Machina actúa como la única interfaz para los usuarios. Todos los especialistas trabajan detrás de escena y son invocados automáticamente según el tipo de solicitud.

matrix@system:~

$ Usuario → Deus Ex Machina → Análisis → ¿Necesita especialista?

↓

$ Sí → Enrutar al especialista apropiado

$ No → Manejar directamente

Principios de Arquitectura

  • 🧠 Inteligencia Raíz + Proyectos Extraídos - El repo raíz es el cerebro
  • 📋 Registro Primero, Extraer bajo Demanda - Solo los proyectos activos se clonan localmente
  • 🎯 Contexto de Proyecto Activo - .context.yaml rastrea el proyecto activo
  • 🤖 Agentes Nativos Devin - Cada agente usa la estructura nativa de Devin
  • 📁 Estado Basado en Archivos - Memoria y estado son archivos, no bases de datos

Tutorial Interactivo: Debuggear un Bug Real

Aprende cómo usar Matrix en un caso de uso real: debuggear un bug en tu proyecto. Este tutorial te guía paso a paso a través del flujo completo.

Paso 1: Configurar tu Proyecto

matrix@setup:~

$ ./bin/matrix add "mi-api-rest" "/ruta/a/mi-api-rest"

✓ Proyecto registrado

$ ./bin/matrix select "mi-api-rest"

✓ Proyecto activo seleccionado

✓ Symlink _brain creado

$ cd /ruta/a/mi-api-rest

✓ Navegado al proyecto

Paso 2: Identificar el Problema

Imagina que tienes un bug en el endpoint de login: los usuarios no pueden autenticarse cuando usan caracteres especiales en su contraseña.

matrix@debug:~

$ /deus-ex-machina "Tengo un bug en el login, los usuarios no pueden autenticarse cuando usan caracteres especiales en la contraseña"

Paso 3: Enrutamiento Automático a Smith

matrix@routing:~

$ Deus Ex Machina: "Detecto palabras clave de bug. Voy a enrutar esto a Smith, nuestro especialista en detección de bugs."

$ Smith: "Analizando el problema de autenticación con caracteres especiales..."

Paso 4: Análisis del Bug por Smith

smith@analysis:~

$ Smith: "Revisando el código de autenticación..."

→ Buscando archivos de login

→ Analizando validación de contraseña

→ Revisando sanitización de input

$ Smith: "Causa raíz encontrada: La función de validación no escapa caracteres especiales antes de comparar con la base de datos."

Paso 5: Coordinación con Trinity para la Solución

trinity@solution:~

$ Trinity: "Diseñando la solución para el bug de caracteres especiales..."

→ Implementar función de escape seguro

→ Agregar tests unitarios para casos edge

→ Validar con diferentes juegos de caracteres

$ Trinity: "Solución implementada. Coordinando con Architect para revisión."

Paso 6: Revisión de Código por Architect

architect@review:~

$ Architect: "Revisando el código de escape de caracteres..."

→ Verificando best practices de seguridad

→ Validando performance de la función

→ Revisando cobertura de tests

$ Architect: "✓ Código aprobado. La solución sigue las mejores prácticas."

Paso 7: Verificación de Seguridad por Sentinel

sentinel@security:~

$ Sentinel: "Analizando vulnerabilidades potenciales..."

→ Verificando prevención de SQL injection

→ Validando que no se introduzcan nuevos vectores de ataque

→ Revisando manejo de errores sin exponer información sensible

$ Sentinel: "✓ Seguridad validada. No hay vulnerabilidades introducidas."

Paso 8: Entrega de Resultados

deus-ex-machina@delivery:~

$ Deus Ex Machina: "¡Bug resuelto! Aquí está el resumen:"

→ Causa raíz: Falta de escape de caracteres especiales

→ Solución: Función de escape seguro implementada

→ Tests: 15 tests nuevos agregados

→ Seguridad: Validado por Sentinel

→ Calidad: Aprobado por Architect

$ Deus Ex Machina: "¿Quieres que cree un checkpoint para este progreso?"

Paso 9: Crear Checkpoint

matrix@checkpoint:~

$ ./bin/matrix checkpoint "Bug de login con caracteres especiales resuelto - función de escape implementada y validada"

✓ Checkpoint creado

✓ Progreso guardado en brain/state/checkpoints/

Resumen del Flujo

1. Configuración: Registrar y seleccionar proyecto 2. Problema: Usuario describe el bug en lenguaje natural 3. Enrutamiento: Deus Ex Machina detecta keywords y enruta a Smith 4. Análisis: Smith encuentra la causa raíz 5. Solución: Trinity diseña e implementa la solución 6. Revisión: Architect valida calidad y best practices 7. Seguridad: Sentinel verifica no hay vulnerabilidades 8. Entrega: Deus Ex Machina presenta resultados consolidados 9. Checkpoint: Se guarda el progreso para futuras sesiones Tiempo total: ~15-20 minutos (vs horas de debugging manual) Coordinación: 4 especialistas trabajando en conjunto Calidad: Validado por múltiples especialistas

Consejos Pro

  • 🎯 Se específico - Cuanto más detalle des, mejor será el análisis
  • 📋 Usa checkpoints - Guarda progreso después de cada tarea importante
  • 🤝 Confía en la coordinación - Los especialistas se comunican entre sí
  • 📝 Revisa los resultados - Siempre valida el trabajo entregado
  • 🔄 Itera si es necesario - Puedes pedir ajustes en cualquier momento

Comandos CLI

El CLI de Matrix proporciona estas operaciones esenciales:

Gestión de Proyectos

# Agregar un proyecto al registro ./bin/matrix add "mi-proyecto" "/ruta/al/proyecto" # Seleccionar proyecto activo (crea symlink _brain) ./bin/matrix select "mi-proyecto" # Ver estado del sistema ./bin/matrix status # Listar todos los proyectos registrados ./bin/matrix list # Limpiar selección actual ./bin/matrix deselect # Escribir un checkpoint ./bin/matrix checkpoint "Completado fix para cliente X"

Invocar al Agente Maestro

# Desde el directorio Matrix /deus-ex-machina "Arregla el bug de login en el módulo de autenticación" # Desde un proyecto activo (con symlink _brain) /deus-ex-machina "Analiza el problema de rendimiento en la API" # Ejemplos de solicitudes /deus-ex-machina "Planifica la migración a la nueva arquitectura" /deus-ex-machina "Investiga cómo funciona el sistema de caché" /deus-ex-machina "Diseña la arquitectura para el nuevo servicio" /deus-ex-machina "Revisa este código y dime qué opinas"

Ejemplo de Uso Completo

# 1. Agregar tu proyecto ./bin/matrix add "mi-proyecto" "/ruta/a/mi-proyecto" # 2. Seleccionarlo (crea symlink _brain) ./bin/matrix select "mi-proyecto" # 3. Navegar a tu proyecto cd /ruta/a/mi-proyecto # 4. Invocar al agente maestro /deus-ex-machina "Necesito arreglar un bug en el flujo de autenticación" # 5. Escribir un checkpoint cuando termines ./bin/matrix checkpoint "Arreglado bug de autenticación - usuario puede hacer login"
